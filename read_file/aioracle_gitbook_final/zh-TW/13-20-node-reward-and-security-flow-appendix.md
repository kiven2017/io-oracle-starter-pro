# 20. Node Reward and Security Flow Appendix｜節點獎勵與安全流程附錄

對 AIOracle 而言，節點激勵的關鍵不是單純獎勵算力，而是獎勵「可驗證、可被網路採納的有效工作」。因此，工作量、質押、質量分數與懲罰規則必須形成聯動，而不能只以請求數量作為指標。

節點安全也不能只依賴單一機制。AIOracle 的設計是把結果共識、抽檢、質量衰減、隨機重算、TEE 執行與計算證明結合起來，使節點即使在面對高價值請求時，也難以通過低成本投機手段長期獲利。
